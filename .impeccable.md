# Impeccable Design Context

## Design Context

### Users
Internal admin users (Lissome staff) managing payroll commission calculations. They use this tool periodically to upload payroll workbooks, configure run settings, and trigger commission calculations. Context is focused, task-oriented work — users need clarity, confidence in what they're doing, and quick feedback on actions.

### Brand Personality
**Professional, precise, composed.**
The interface should feel like a premium internal tool — not flashy, but polished and intentional. Every element earns its place.

### Aesthetic Direction

**Visual tone:** HBO-inspired dark premium aesthetic adapted for a functional admin tool. Dark, high-contrast, minimal, authoritative.

**Theme:** Dark mode primary. Black and charcoal backgrounds with high-contrast white/light text.

**Reference:** [hbo.com](https://hbo.com) — specifically:
- Dark palette with `#000000`, `#0f0e13`, charcoal grays against clean white text (`#f6f6f6`, `#FFFFFF`)
- Uppercase headings with generous letter-spacing and clean font hierarchy
- Minimal dark buttons with subtle hover transitions (`background-color 0.5s ease-in-out`)
- Generous whitespace and breathing room between sections

**Anti-references:** Generic Bootstrap admin templates, overly colorful dashboards, Material Design defaults.

### Color Palette (derived from HBO)

| Role | Value | Usage |
|------|-------|-------|
| Background | `#000000` | Page background |
| Surface | `#0f0e13` | Cards, panels, elevated surfaces |
| Border | `#4E505B` | Subtle borders, dividers |
| Text primary | `#FFFFFF` | Headings, primary content |
| Text secondary | `#f6f6f6` | Body text, descriptions |
| Text muted | `#9ca3af` | Labels, metadata, hints |
| Accent | `#4E505B` | Hover states, secondary actions |
| Success | `#22c55e` | Confirmation messages, success states |
| Danger | `#ef4444` | Error messages, destructive actions |
| Focus ring | `rgba(13,110,253,0.25)` | Focus outlines for accessibility |

### Typography

- **Font stack:** `"Helvetica Neue", Helvetica, Arial, sans-serif`
- **Headings:** Uppercase, bold (700), generous letter-spacing (0.05–0.1em)
- **Body:** Regular (400), 0.875rem–1rem, line-height 1.5
- **Labels:** Medium (500), 0.75rem–0.875rem, uppercase optional
- **Hierarchy through size and weight, not color variety**

### Spacing Scale

Use a consistent 4px/0.25rem base:
- `0.25rem` (4px), `0.5rem` (8px), `0.75rem` (12px), `1rem` (16px), `1.5rem` (24px), `2rem` (32px), `3rem` (48px)
- Generous padding inside cards/panels: minimum `1.5rem`
- Section spacing: `2rem`–`3rem` between major sections

### Component Guidelines

**Buttons:**
- Dark background (`#000` or `#0f0e13`), white text, uppercase, border-radius `0.3rem`
- Border: `1px solid #4E505B`
- Hover: background shifts to `#4E505B`, transition `0.5s ease-in-out`
- Padding: `0.5rem 1rem` minimum

**Cards/Panels:**
- Background: `#0f0e13` on `#000000` page
- Border: `1px solid #4E505B`, border-radius `0.3rem`
- Padding: `1.5rem`
- Subtle shadow: `rgb(78 80 91 / 20%) 0 4px 16px`

**Form inputs:**
- Dark background matching surface color
- Light border (`#4E505B`), white text
- Focus: blue ring `rgba(13,110,253,0.25)`

**Transitions:**
- All interactive elements: `transition: all 0.3s ease-in-out` minimum
- Background color changes: `0.5s ease-in-out`

### Design Principles

1. **Dark canvas, light content** — Black backgrounds create focus; white text commands attention. Use color sparingly and only for semantic meaning (success, error, action).

2. **Typographic hierarchy over decoration** — Communicate importance through font size, weight, and case transforms. Avoid icons, badges, or colors where text weight and spacing suffice.

3. **Generous breathing room** — Every element needs space. When in doubt, add more padding. Dense admin UIs feel stressful; this tool should feel composed.

4. **Subtle motion, not animation** — Transitions signal interactivity (hover, focus) but never distract. No bounces, no slides, no loading spinners with personality. Ease-in-out, 0.3–0.5s.

5. **Confidence through clarity** — Destructive or important actions (Update Talenox, Run Commission) must be visually distinct and require deliberate interaction. The UI should make the user feel certain about what will happen.

### Tech Stack Context

- **HTML + Tailwind CSS** (CDN v2.2.19) + **HTMX** for interactivity
- Plain JavaScript (no framework)
- Express.js server serving static files from `public/`
- Two pages: `index.html` (main commission config/run) and `staffhurdle.html` (staff hurdle config)
